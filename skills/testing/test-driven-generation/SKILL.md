# Test-Driven Generation

## Purpose

Generate implementation from a test suite (TDD approach).

## Workflow

1. **Input**: Test suite (failing or with expectations).
2. **Analyze**: Extract expected behavior from tests.
3. **Generate**: Produce implementation that satisfies tests.
4. **Validate**: Run tests to confirm.
5. **Iterate**: Fix until all tests pass.

## Output

- Implementation code
- Test run result
- Notes on design decisions
