# API Design Assistant

## Purpose

Help design clean, consistent public APIs following language/framework best practices.

## Workflow

1. **Input**: API surface (signatures, docs) or intended behavior.
2. **Review**: Check naming, consistency, discoverability, error handling.
3. **Suggest**: Improvements for clarity, consistency, and usability.
4. **Document**: Recommend docstrings and examples.

## Output

- API design recommendations
- Naming and consistency suggestions
- Example usage or docstrings
